## C/C++ 中的指针定义

学过的都知道，*C/C++* 中的指针声明常常令人难以理解，再搭配上 `const` 的左结合设计更是令人头大，比如下面这个来自 [cdecl.org](https://cdecl.org/?q=int+%28*%28*foo%29%28void+%29%29%5B3%5D) 的例子：

```c
// declare foo as pointer to function (void) returning pointer to array 3
// of int
int (*(*foo)(void ))[3]
```

我们就不去扯这些难的东西了，这里就讨论一下指针指向的数据与指针本身的不可变性，也就是 `const` 的修饰问题. 我们知道 `const` 具有左结合性，即默认使其左边的对象不可变，若找不到，再修饰其右边的对象. 下面几个例子就是我们今天讨论的对象：

```c
// 一个指向不可变数据的指针
const int *p;  // 等价于 int const *p;
// 一个指向的目标不可变的指针
int *const p;
// 一个指向不可变数据的不可变指针
const int* const p; // 等价于 int const * cosnt p;
```

上面的例子中未注释的版本都是比较常见的写法. 当然你自己想怎么写随意了，一切按照你便于理解的方式写就行了. 另外，我上面把常量称作“不可变”，也是便于之后我们和 Rust 联系起来. 接下来我们先给上面的三条语句举一些例子.

:::warn
下面的代码采用 `C` 编写，在 `ArchLinux/gcc (GCC) 15.2.1 20260209` 下编译运行. 且为了演示方便，省略了 `main` 函数的声明.
:::

“一个指向不可变数据的指针”，就是说指针本身的值（即指针指向的目标）是可以修改的，但是不能通过指针去修改指向目标本身的内容. 我们通过下面的代码来理解这个性质，并画一个图来形象地理解：

```c
// declare p as pointer to const int
const int *p;
int a = 1234;
p = &a;
a = 1494;
// *p = 1494;  // Compiling Error!!!
//    ^~~~~~ error: assignment of read-only location ‘*p’
```

也就是说我们可以通过变量 `a` 来修改它的值，因为我们并没有声明 `a` 是常量，但是不能通过 `*p` 来修改 `a` 的值，因为我们声明了不可变动 `p` 指向的数据. （实际上这里的 “不可变”准确来说应该是不可通过 `p` 来“变动”，我们仍然可以通过 `a` 来修改数据，甚至再声明一个指针来修改数据）.

下图中，我们使用 `<变量名><地址(蓝)><值(绿，方框)>` 来表示一个变量.

![20260413-bd043b4b10afb800.png](./images/20260413-bd043b4b10afb800.png)

接下来，为了演示指针 `p` 的值可以修改：

```c
const int b = 5678;
p = &b;
// b = 7796;  // Compiling Error!!!
//   ^~~~~~ error: assignment of read-only variable ‘b’
```

如图所示，我们修改了 `p`，使其指向了 `b`！需要注意的是，这里我们的 `b` 声明为 `const int b`，因此无法通过 `b = ?` 的方式来直接修改 `b` （还有其他办法吗？我们后面会提到）.

![20260413-31d8ec626e2f5494.png](./images/20260413-31d8ec626e2f5494.png)

“一个指向的目标不可变的指针”，就是说指针本身的值（即指向的目标）不可改变，但是可以通过指针去修改指向目标本身的内容，同样看代码：

```c
int *const p1 = &b;
// p1 = a;
*p1 = 7796;
```

![20260413-dc34142bf2b83da5.png](./images/20260413-dc34142bf2b83da5.png)

通过以上代码，我们发现一个有意思的点，不仅仅是可以 `p1` 指向的目标可以修改其值，更重要的是我们修改了一个常量的值！原因很简单，因为在 `C` 中栈内存的权限一般是可读可写的，而我们的 `b` 定义在 `main` 方法中，即放在栈内存下，因此是可以读写的. `const` 只能在编译器层面进行限制.

当然，这个代码会引发编译警告，因为这是一个未定义行为，但确实带给我们一些启发，一些内存泄漏的风险，人们常常因为指针引发的一些错误可能就有部分是因为这个未定义行为！

---

到此为止了吗？不！如果你把 `const int b = 5678;` 这条语句放在 `main` 函数外呢？`b` 成为了一个全局变量！也就是全局的常量，根据计算机系统基础的知识（x86），全局常量编译后会被放在 `.rodata` 中，这块内存是只读的，因此，当我们编译后（仍然有警告，但是还是没有编译错误），一旦运行就会得到段错误（因为我们访问了只读的段）.

算是一点点小小的拓展.

---

我们回到第三个例子，“一个指向不可变数据的不可变指针”，即 `const int* const p`，这是一个即不可以修改指向的目标，也不可以修改目标本身数据的指针，我们只能读它. 比较简单，就不给图了. 一个代码示例：

```c
const int* const p2 = a;
```

:::spoi 全部代码汇总

```c
#include <stdio.h>

const int b = 5678;
int main() {
    const int *p;
    int a = 1234;
    p = &a;
    a = 1494;
    // *p = 1494;  // Compiling Error!!!
    p = &b;
    // b = 7796;  // Compiling Error!!!
    int *const p1 = &b;
    // p1 = a;
    // *p1 = 7796;

    const int* const p2 = &a;
}
```
:::

## Rust？

接下来我将要提到的这种写法其实不太常见，就是变量本身 `mut` 的同时引用也可以 `mut`:

```rust
let mut a = 1u32;
let mut b = 2u32;
let mut v: &mut u32 = &mut a;
v = &mut b;
```

是的，我们让变量 `v` 可以更换绑定不同的引用，同时也可以修改引用的数据，但是在 Rust 中，用一个 `&mut` 去修改一个 immutable 的变量就不可能了，在编译阶段就会把我们拦下来.

我们可以分别给出 *C* 中的指针在 *Rust* 下的近似实现：

```rust
// int *p;
let mut p: &mut i32;
// const int *p;
let mut p: &i32;
// int *const p;
let p: &mut i32;
// const int * const p;
let p: &i32;
```

算是一点小小的思考，以前一直没有注意到在 *Rust* 中变量可变和类型为可变引用实际上是不同的概念这一点. 进而引申出了 *C/C++* 和 *Rust* 在设计上的区别.
